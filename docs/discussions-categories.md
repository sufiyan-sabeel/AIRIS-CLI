# GitHub Discussions Categories

Recommended categories for GitHub Discussions when enabled:

## Categories

| Category | Description | Purpose |
|----------|-------------|---------|
| **Announcements** | Official updates from maintainers | Release notes, roadmap updates |
| **General** | Open discussion about AIRIS | Feature ideas, usage questions |
| **Ideas** | Feature requests and suggestions | Community-driven roadmap input |
| **Q&A** | Questions that have clear answers | Searchable knowledge base |
| **Show and Tell** | Share what you built with AIRIS | Extensions, workflows, projects |
| **Development** | Contributor discussion | Architecture decisions, PRs |
| **Android/Termux** | Mobile-specific discussion | Android tips, issues, workarounds |

## Setup

To enable GitHub Discussions:
1. Go to repository Settings
2. Scroll to Features section
3. Check "Discussions"
4. Create categories matching the table above

## Guidelines

- Use Issues for bugs and feature requests (with templates)
- Use Discussions for questions, ideas, and community interaction
- Mark accepted answers in Q&A
- Pin important announcements
