# AIRIS Memory

Optional project memory for AIRIS sessions. Enable it only when durable cross-session context is useful.

Suggested files:

- `project-brief.md`: goals, scope, non-goals
- `product-context.md`: why the project exists and user expectations
- `active-context.md`: current focus, recent decisions, next steps
- `system-patterns.md`: architecture and recurring implementation patterns
- `tech-context.md`: stack, commands, setup constraints
- `progress.md`: completed work, remaining work, known issues

Keep memory concise. Do not store secrets or credentials.
