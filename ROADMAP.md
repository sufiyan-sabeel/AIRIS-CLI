# AIRIS Roadmap

This document outlines the planned development path for AIRIS. Features are organized by release milestone.

## v0.80 -- Stable TUI, Adaptive Brain, Permissions, Doctor

**Status: In Progress**

- Stable terminal user interface with all 14 themes
- Adaptive Brain for context-aware responses
- Project trust and permission system
- `airis doctor` runtime health diagnostics
- `airis ship` full-lifecycle development workflow
- Verified Autonomy mission contracts and evidence
- Session management (save, resume, fork, export)
- Multi-provider model routing

## v0.85 -- Model Router, Privacy Firewall, Repository Intelligence

**Status: Planned**

- Intelligent model router: automatic provider/model selection based on task type
- Privacy firewall: local-first data handling with configurable remote access policies
- Repository intelligence: codebase-aware context building, dependency graph analysis
- Enhanced `airis ship` with automatic test generation
- Cross-session learning and memory persistence
- Improved error recovery and retry logic

## v0.90 -- Visual Verification, Android Bridge, Local Automation

**Status: Planned**

- Visual verification: screenshot capture and comparison for UI changes
- Android Bridge: deeper integration with Android APIs beyond shell commands
- Local automation: scheduled tasks, file watchers, and event-driven workflows
- `airis ship` visual project support (screenshots, UI diff review)
- Collaborative sessions: multi-user real-time editing
- Plugin marketplace for community extensions

## v1.0 -- Polished Free Open-Source Release

**Status: Planned**

- Production-ready stability across all platforms
- Comprehensive documentation and tutorials
- Performance optimization for large codebases
- Accessibility improvements for terminal UI
- Full test coverage for core functionality
- Community-driven extension ecosystem

## v2.0 -- Optional AIRIS Cloud and Team Services

**Status: Future**

- Optional cloud sync for sessions and configuration
- Team collaboration features
- Shared project contexts and memory
- Enterprise deployment and management tools
- Custom model fine-tuning integration

---

## Versioning Policy

AIRIS follows lockstep versioning: all packages share one version number. Patch releases include fixes and minor additions. Minor releases include breaking changes. There are no major version bumps during the v0.x series.

## Contributing to the Roadmap

Feature requests and roadmap discussions happen in [GitHub Issues](https://github.com/sufiyan-sabeel/AIRIS-CLI/issues) and [Discord](https://discord.com/invite/nKXTsAcmbT). The roadmap is maintained by the core team and updated as priorities evolve.
